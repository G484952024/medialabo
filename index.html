プロジェクト課題について

グルメ情報の検索についてのWebページを作成しました。

Webページでジャンルを選び検索ボタンを押して検索できるよにしました。

選んだ検索結果についての表示が行われ、店舗の名前、アクセス情報、住所、予算、キャッチコピー、ジャンル、営業日時、最寄り駅、サブジャンルの名前を表示します。

検索件数を表示したり、カードの中に情報を入れることでどこまでが1つの店舗の表示なのかを分かりやすくしました。

プロジェクト課題で作成したページ:プロジェクト課題

数当てゲーム

1~10の乱数で決められた数を答えとして,ユーザーは答えを予想して3回まで整数を入力する.

予想が間違っていたら，予想より答えが大きいのか小さいのかを表示する．

3回までで答えを当てられたらユーザーの勝ち,3回までに答えを当てることができなければ負け。

テキストボックスに整数を入力して回答のボタンを押すと正解か不正解かの判定が行われる。

予想回数が表示され、正解すると正解のメッセージが表示される。

3回間違えると答えが自動的に表示され、4回以上回答するとゲームが終わっているという表示出る。

デバック用の答えがコンソールに表示されるようにする。

数当てゲームのページ:課題3-1:数当てゲーム

各授業回の学習事項

第1回 WWW と HTML

HTMLは様々な種類の要素で作られ、その要素はタグで囲まれる。

p要素は段落、strong要素は強調の働きがある。

たくさんの種類があるHTMLの要素を使ってWebページの構造を記述していく。

演習の成果: 演習1-4: HTMLのさまざまな要素

第２回 CSSによるWebページの装飾

CSSとはどうゆうものなのかを知り、CSSの種類や調べ方を学習する。

CSSを使ってWebページのレイアウトや装飾をする。

レイアウトや装飾の記述の仕方を理解し、レイアウトや色使いを考えて実際にWebページを作る。

演習の成果: 演習2-3: ウェブブラウザ説明ページの装飾

第３回 JavaScript の基本

JavaScriptはWebサイトでボタンを押したり、動きを持たせる役割をしている。

JavaScriptはコンパイルが必要なく、一行ずつ読み取り実行されるので、コンソールで確認しながらプログラムを書く。

JavaScriptではエラーが発生しない場合があり、undefinedと表示されることがある。

演習の成果:演習3-6:オブジェクトの参照

第４回 DOM と JavaScript

input要素は主にテキストボックスの作成に必要な要素や属性が記述される。

button要素はボタンをクリックしたときに次のアクションを起こす時に使われる。

JavaScriptのプログラムで要素を検索する時は、document.querySelector()で要素を1つずつ、
document.querySelectorAll()ですべての要素を検索することができる。

演習の成果:演習4-10:足し算ページ

第５回 イベント処理

DOMは要素の作成や要素の編集、削除も行うことができる。

要素を新しく作成するには、要素のオブジェクトを新規作成し、その要素を入れる場所を決めるためにHTMLの中の要素を検索する。

検索した要素の直後や内部に新しい要素を置くことができる。

演習の成果:演習5-7:DOMプログラミング(2)

工夫点、苦心点、強調点、感想

今回のメディア実験の授業でWebサイトの仕組みを知ることができました。

WebサイトはHTMLとCSSとJavaScriptでできていることやそれぞれについての記述の仕方やどう動いているのかどんな役割があるのかを学ぶことができました。

自分で実際にWebページを作ることはとても分かりやすくWebページの動きや仕組みを知るきっかけになったと思います。

画像やリンクをHTMLを使ってページに表示したり、数当てゲームでは、Webページで自分が書いたJavaScriptのプログラムがどう動くのかを確認できてとても分かりやすかったです。

工夫点は、project課題の表示結果をカード風の表示にしたことです。

普段からwebサイトを使っていて知りたい情報が1つずつ分かりやすくまとまっていれば読みやすいし、情報を楽に入手することができるのでは？と感じたので表示する店舗ごとの情報をカードの中にまとめて表示してみました。

色なども黄色など薄い色を選び中の文字が見づらくならないものを選びました。検索の仕方は選択式にしました。

打ち込み式も良いとは思ったのですが、打ち込むより選択する方が私的には楽だと感じたので選択式にしました。

他には、検索された結果が何件目なのかを表示できるようにしました。

それによってどこまでが1つの店舗についての表示なのかが分かりやすくなるのでここは強調したかった点です。

検索結果が読み出されるたびに件数の数が増えるようにfor文を使って表示する部分に苦労しました。

JavaScriptのプログラムを書くときに頭の中がしっかり整理するのが難しくてプログラムがわかりにくく書かれることが多く苦労しました。

2回目以降に検索すると店名だけが消えるという事件が起きたのですがプログラムの内容や意味を1つずつ理解していくときれいに正しくプログラムを書くことができました。

HTML、CSS、JavaScriptの中ではJavaScriptが一番理解するのも使うのも難しいと感じました。

はじめてJavaScriptについて学んで、実際に使って見たときJavaと表示の仕方などが全然違ったので理解して自分の思いどおりに書くのが難しく、まだ完全に使いこなせるわけではありませんがたくさん使って慣れて行きたいと思います。

これからも他に自分の作ってみたいWebページをHTMLやCSS、JavaScriptを使ってもっと自分の思いどおりに動かしてみたいと思いました。
